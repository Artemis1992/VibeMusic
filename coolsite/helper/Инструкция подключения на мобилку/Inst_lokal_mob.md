# Доступ к локальному сайту Django с телефона

**Цель**  
Обеспечить доступ к локальному Django-сайту с телефона, находящегося в одной сети Wi-Fi с компьютером.

---

### 1. Получение IP-адреса компьютера

- Открыть **командную строку (cmd)**.  
- Выполнить команду:  
### *bash - `win+R` - `cmd.exe`*
  ```
  
  ipconfig
  ```
- Найти раздел Wi-Fi и строку IPv4-адрес.

```
IPv4-адрес . . . . . . . . . . : 192.168.100.33
```

*Обоснование: Телефон должен знать локальный IP-адрес компьютера, чтобы обратиться к серверу.*

### 2. Обновление файла settings.py

- Открыть файл `settings.py`.

- В списке `ALLOWED_HOSTS` добавить найденный IP:

```
ALLOWED_HOSTS = ['localhost', '127.0.0.1', '192.168.100.33']
```
*Обоснование: Django должен разрешать обращения к сайту по локальному IP.*


### 3. Запуск сервера разработки

* Выполнить команду:
```
python manage.py runserver 0.0.0.0:8000

```
* Альтернатива (если возникает ошибка `not a valid port`):

```
python manage.py runserver 192.168.100.33:8000
```
*Обоснование: Сервер будет доступен по всем интерфейсам, включая локальную сеть.*

### 4. Подключение телефона

* Убедиться, что телефон и ПК находятся в одной сети Wi-Fi.

* На телефоне в браузере открыть:

```
http://192.168.100.33:8000/
```
*Обоснование: Теперь локальный сайт доступен с телефона по IP-адресу компьютера./*
