# redirect()

## Описание
Функция `redirect` из `django.shortcuts` создаёт HTTP-перенаправление на указанный URL, маршрут или объект модели. В `vibemusic` используется для перенаправления после входа, выхода, создания поста или комментария (например, в `logout_user`, `add_comment`).

### Особенности
- **Гибкость**: Поддерживает именованные маршруты, прямые URL и объекты моделей.
- **Типы перенаправления**: Временное (302) или постоянное (301, через `permanent=True`).
- **Использование в vibemusic**: Перенаправление на главную страницу или страницу поста после действий.

### Таблица
| Аспект                    | Описание                                                                 |
|:--------------------------|:------------------------------------------------------------------------|
| **Назначение**            | Перенаправление пользователя на другой URL или маршрут.                  |
| **Параметры**             | `to` (имя маршрута, URL или объект модели), `*args`, `**kwargs` (для маршрутов), `permanent` (булево, для 301/302). |
| **Пример использования**  | ```python<br>from django.shortcuts import redirect<br>import logging<br>logger = logging.getLogger(__name__)<br>logger.info("Redirecting to login")<br>return redirect('login')<br>``` |
| **Возвращаемый объект**   | `HttpResponseRedirect` или `HttpResponsePermanentRedirect`.              |
| **Возможные исключения**  | `Resolver404` (если маршрут не найден).                                 |
| **Примечания**            | Используется для управления потоком после действий пользователя.         |
| **Пример в vibemusic**    | В `logout_user`: `return redirect('login')`.                             |
| **Логирование**           | Логировать перенаправления (`logger.info(f"Redirected to {to}")`).      |
| **Сценарии использования**| Перенаправление после входа, выхода, создания поста/комментария.         |
| **Рекомендации по безопасности** | Проверять валидность URL при использовании пользовательского ввода.      |
| **Типичные коды ответа**  | 302 (временное перенаправление), 301 (постоянное, если `permanent=True`). |
